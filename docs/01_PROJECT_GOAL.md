## 0. 一句话

> 多方向扫描的能力是真的,但它被不被使用,取决于 SSM 承担了多少空间混合的工作。
> 现有论文把增益归给了方向,而增益主要来自多分支容量。

---

## 1. 起点:一个被广泛复述但从未被因果检验的机制主张

VMamba、Vim、GroupMamba、UltraLight VM-UNet 等一系列 Vision Mamba 工作都采用多方向
扫描,并给出同一个机制解释:**"补充方向信息"**——单向扫描会丢失某些方向上的空间依赖,
加入更多方向可以补上。

这个主张有两个可分离的部分:

| | 主张 | 是否被检验过 |
|---|---|---|
| **A. 能力主张** | 单向扫描确实会丢失方向信息 | **从未被隔离检验** |
| **B. 归因主张** | 多方向的性能增益来自补上了这个缺口 | **从未被因果隔离** |

现有论文只报告了"加方向 → 涨点",这同时兼容 B 和另外几种解释(容量、路径多样性、
正则化)。**没有一篇做过参数匹配的因果隔离。**

---

## 2. 文献定位(重要:本节纠正旧版的错误定位)

### 2.1 我们的批评不是新的

Zhu et al.(arXiv 2405.08493, *Rethinking Scanning Strategies with Vision Mamba in
Semantic Segmentation of Remote Sensing Imagery*)已经指出:多方向扫描的假设缺乏
定量验证;VMamba 的四方向在 ImageNet 上仅比单向高 0.3%,考虑训练波动,这一提升
不足以证实多方向扫描的有效性。他们系统消融了 22 种扫描顺序,结论是航拍分割中单一
左到右扫描即可。

**必须在 Related Work 中主动引用并划清边界。** 他们做的是经验比较(哪种顺序涨点多),
没有做容量匹配对照,也没有给出机制解释。

### 2.2 GroupMamba 自己的消融接近我们的核心对照

GroupMamba(arXiv 2407.13772)Table 4:

| 扫描方向 | 参数量 | Top-1 |
|---|---|---|
| D1 | 23M | 82.9 |
| D1, D2 | 23M | 83.1 |
| D1–D4 | 23M | 83.3 |

分组数固定为 4,参数量恒定。因此 "D1" 一行实质上就是**四组全扫 D1**,即我们的
`channel_same_row_4`;"D1–D4" 即 `channel_real_4dir`。

**方向的净贡献是 +0.4pp。** 他们把它解释为"四个方向捕获更丰富的空间线索",
但从未追问为何如此之小,也从未检验模型是否真的在使用扫描顺序。

**含义:我们的 channel-split 实验将复现他们的数字。这不是缺陷 —— 它验证了我们的
设置,而新增的 2×2 析因(见 §4.3)是他们没有的。**
**待办:精读 GroupMamba,确认 D1 一行确为"四组全 D1"而非"单组全宽 D1"。
这直接决定我们的 channel-split 是复现还是新发现。**

### 2.3 "方向价值随序列长度上升"已被观察到

ZigMa(arXiv 2403.13802, §4.2)报告:zigzag 方案从 1 增至 8 时增益持续提升,
且 Zigzag-1 与 Zigzag-8 的相对增益在更高分辨率(512×512,即更长 token 序列)下
更为显著。

Arcee(arXiv 2511.11243)在 k ∈ {1,2,4,8} 上观察到三种 regime。

**这是我们 scan-load 发现的先声。** 但它们是经验观察,没有因果隔离,也没有把
"序列长度"与"局部聚合分工"分开。

### 2.4 文献是互相矛盾的 —— 而这正是我们的入口

| 论文 | 场景 | 结论 |
|---|---|---|
| Zhu et al. | 航拍分割 | 单向即可,方向不重要 |
| GroupMamba | ImageNet 分类 | 四方向仅 +0.4pp |
| ZigMa | 高分辨率扩散 | 方向多样性持续获益,分辨率越高越明显 |
| Flatten Wisely | 脑 MRI 分割 | 扫描顺序显著影响 Dice |

**没有人解释为什么这些结论互相打架。**

## 3. 本文的主张

> 这些不是矛盾,而是同一根轴上的不同位置。
>
> **方向的价值由"扫描负载"(scan load)决定** —— 即空间混合这项工作在
> patch embedding 卷积与 SSM 之间如何分工。
>
> - 卷积承担了空间聚合、序列短(ImageNet 分类、下采样后的航拍图)→ **方向能力闲置**
> - SSM 必须独自搬运空间信息、序列长(高分辨率扩散、密集医学分割)→ **方向被激活**

配套的机制账:

1. **能力真实存在。** 受控 carry 任务中,不匹配的扫描方向使信息传递彻底失败
   (退化为随机猜测)。这不是"稍差",是"完全过不去"。
2. **但自然图像分类几乎不使用它。** 参数匹配后,方向只解释四分支总增益的约四分之一。
3. **原因是分工。** 当卷积把局部信息预先揉好,SSM 不需要做长程搬运,
   单向扫描的有效视野已经足够,增加方向是重复而非补充。

## 4. 使本文区别于现有文献的四件工具

### 4.1 受控 carry benchmark(target-token readout)

合成任务:信息必须沿扫描序列从 source patch 传到 target patch,
readout 只取 target token(**不是 mean-pool** —— 见 03_EVIDENCE_LEDGER §1)。

**这是唯一能隔离"能力"的实验。** 现有文献没有任何等价物。

### 4.2 order_utilization(shuffle 对照)

`acc(row) − acc(shuffle_row)`:打乱 token 顺序后掉多少分。

**这是 scan load 的可操作化度量** —— 它直接回答"模型到底在多大程度上依赖扫描序列"。
现有文献没有任何等价物。

### 4.3 结构性 × 多样性的 2×2 析因(channel-split)

四变体只在"喂给四路 SSM 的 token 顺序"上不同,通道宽度、序列长度、参数量完全一致
(已核对 backbone.py:same_perm=四路同一个固定随机置换;rand_perm=四路四个不同
固定随机置换;两者均存为 persistent buffer,不进参数量)。

|  | 单一(1 种模式 ×4) | 多样(4 种不同模式) |
|---|---|---|
| **结构化行序** | `channel_same_row_4` | `channel_real_4dir` |
| **随机乱序** | `channel_same_perm_4` | `channel_rand_perm_4` |

- **structure 主效应** = {real, same_row} − {rand_perm, same_perm}:保留空间结构 vs 打碎
- **diversity 主效应** = {real, rand_perm} − {same_row, same_perm}:多样 vs 单一重复
- **交互项** = (real − same_row) − (rand_perm − same_perm)
  = **几何方向多样性的收益 − 随机乱序多样性的收益**

**关键是交互项:** 交互 ≈ 0 → 几何方向多样性 = 随机乱序多样性,即多样性红利与
"是否几何"无关,"方向互补"这个概念塌掉;交互 > 0 → 几何方向的多样性有独立于
"任意多样性"的价值,几何结构确实必需。

**GroupMamba 从未做过这个。这是全套设计中唯一能双向判决"方向是不是几何的"的实验。**

### 4.4 CKA 分支相似度分析

排除"表征去相关"作为竞争解释。见 03_EVIDENCE_LEDGER §5。

## 5. 贡献声明(投稿用)

1. **因果隔离协议。** 我们提出一套对照设计,用于分离多方向扫描增益中的
   方向信息、多分支容量与路径多样性三个成分:参数匹配对照(`same_row_4`)、
   顺序打乱对照(`shuffle_row`)、以及局部性 × 多样性的 2×2 析因。

2. **能力存在但未被使用。** 受控 carry 任务中,不匹配扫描退化为随机猜测,
   证明方向信息缺口真实且严重;而在 CIFAR-10 上,四方向相对参数匹配的
   同方向四分支仅提升约 0.6–0.9pp,不足总增益的四分之一。
   **现有论文的增益归因是错的:涨点主要来自多分支容量,而非它们声称的方向互补。**

3. **统一解释现有文献的矛盾。** 方向的价值由扫描负载决定。这解释了为什么
   航拍分割中单向即可(Zhu et al.)、而高分辨率扩散中方向多样性持续获益(ZigMa)。

4. **排除去相关机制。** CKA 分析显示方向带来的分支去相关增益与准确率增益不相关
   (r = −0.22, n=6),排除了"多样性 → 平均降噪"这一竞争解释。

## 6. 明确不主张的东西(claim boundary)

- **不主张"方向无用"。** 方向贡献非零(+0.6~0.9pp,McNemar p=0.0115),
  只是小且被错误归因。过强的否定主张会被 ZigMa / Flatten Wisely 的正面结果驳倒。
- **不主张"我们首先发现多方向增益很小"。** Zhu et al. 和 GroupMamba 的 Table 4
  已经报告了这一点。我们的贡献是**解释和隔离**,不是发现。
- **不主张扫描长度是独立的因果变量。** 在固定图像尺寸下,扫描长度与局部聚合强度
  由同一个旋钮(patch_size)控制,**物理上不可分离**。必须统一称为"扫描负载"。
- **不主张已验证跨模型/跨任务普适性。** 当前证据限于 CIFAR-10 + GRU/Mamba +
  小模型规模。医学分割与高分辨率场景的验证是未来工作。

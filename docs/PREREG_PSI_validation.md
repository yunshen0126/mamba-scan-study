# PSI 验证预注册:置换敏感度作为任务无关扫描负载轴

**撰写时间:2026-07-19,在任何 PSI 数值产生之前。判据部分提交后不得修改。**

---

## 1. 背景与动机

主实验(ledger §7)使用 `order_utilization` 作为扫描负载的度量,并据此主张:

> delta_direction 与 order_utilization 的相关(r=0.947)高于与 log2(L) 的相关(r=0.86),故驱动变量是扫描负载而非序列长度。

**已发现的方法学问题:** 经代码核查,`order_utilization = test_acc(row) − test_acc(shuffle_row)`,是一个分类准确率差值,依赖 logits、mean-pooling readout 与类别标签。而 `delta_direction` 同样是准确率差值。**用一个扫描扰动引起的准确率差去预测另一个扫描扰动引起的准确率差,相关偏高是可预期的,而 log2(L) 作为纯结构变量在此对比中处于不利地位。**

这不构成完全循环——按本文自身的两轴划分,`order_utilization` 属 A 轴(顺序/结构),`delta_direction` 属 B 轴(多方向互补),二者可分离,相关有信息量。但它**不是一个独立于任务表现的负载度量**,而 ledger 目前将其定位为负载轴。此定位需要修正。

同时,批次 E(分割任务析因)要求一个**跨任务可比**的负载轴,以支持"负载匹配条件下比较不同因果结构任务"的核心设计。准确率差值无法跨分类/分割任务通约。

**因此需要一个任务无关的替代度量,并检验其有效性。**

---

## 2. PSI 定义

**Permutation Sensitivity Index (PSI):** 对训练好的模型,输入一批真实图像(不使用标签),在 SSM 堆栈入口施加固定置换 π、出口施加 π⁻¹ 还原,比较该堆栈输出特征与自然顺序下的输出特征:

```
PSI = || F_ord − F_perm ||_F / || F_ord ||_F
```

逐样本计算后对 batch 取均值,再对 8 个独立置换取均值与标准差。

**任务无关性保证:** 仅调用 `forward_features`,不经过分类头,不产生或读取 logits,不使用标签、损失或准确率。已由代码审查确认(`compute_psi.py`, commit fe2e8f0)。

**两种模式:**
- **stack-level(主):** π 施加于 blocks[0] 之前,π⁻¹ 施加于 blocks[-1] 之后。**这与 `shuffle_row` 及 `channel_permutations` 的操纵位置完全一致**,唯一差别是读出方式(特征距离 vs 分类准确率)。**P1–P3 判据一律以 stack-level 为准。**
- **module-level(次):** π/π⁻¹ 施加于单个 block 两侧。用于机制定位(负载敏感性来自第几层),不参与判据判定。

**配置级聚合(事先定死):** 对每个 π,取该配置下所有 branch/group 堆栈 PSI 的算术平均;配置级 PSI 为这些逐 π 均值的算术平均。此规则在看到任何数据前定死,不得事后调整。

**取值范围限定:** 负载轴取值**只能来自中性模型**——full-branch 的 `row`,channel-split 的 `same_row_4`。`real_4dir` 等经过方向操纵的变体是处理组,在其上测得的负载已被处理污染,不得用作负载轴取值(可另行报告作为附带观察,标注 `is_canonical=False`)。

## 3. 预注册判据

### P1 — 单调性(有效性的必要条件)

在中性模型上,配置级 stack-level PSI 随 grid 8→16→32 **单调递增**,GRU 与 Mamba 两个 backbone 均成立。

**判定:** 5 seed 的均值严格单调即通过。若任一 backbone 出现非单调,P1 不通过。

**若 P1 不通过:** PSI 不反映扫描负载,不得用作负载轴。退回方案二(跨 token 影响范围度量)重新设计,或在批次 E 中改用序列长度 L 作为较弱的代理并明确承认其局限。

### P2 — 初始化对照(排除架构平凡解释)

同时在**随机初始化的未训练模型**(`--at-init`)上计算 PSI。

**判定:** 训练后模型的 PSI 单调性应**显著强于**随机初始化模型。具体操作化为:训练后模型的 PSI(grid32) / PSI(grid8) 比值,大于随机初始化模型的对应比值。

**若 P2 不通过(两者比值相当):** 说明 PSI 主要由架构与序列长度决定,而非训练中学到的扫描负载利用。此时 PSI 与 log2(L) 高度共线,作为"负载"轴的解释力被削弱。**此情形必须如实报告**,并在论文中将 PSI 表述为"结构性负载代理"而非"学得的负载利用度"。

### P3 — 重做负载 vs 长度的解离(无通过门槛)

以 stack-level PSI 替代 `order_utilization`,重算其与 `delta_direction` 的相关,并与 log2(L) 和 `delta_direction` 的相关对比。使用主实验 full-branch 的 5-seed 数据(6 个配置:2 backbone × 3 grid)。

**本条不设通过门槛。无论结果如何都如实报告。** 三种可能:

- **PSI 相关仍明显高于 log2(L):** 原主张从"半循环"升级为"由任务无关度量独立证实"。ledger §7 相应加强,并说明原 `order_utilization` 版本作为任务相关的辅助证据保留。
- **PSI 相关与 log2(L) 相当:** 说明原 r=0.947 中含有度量同源的成分。ledger §7 必须收紧:不再主张"负载而非长度是驱动变量",改为"在现有 3 个负载点上,负载与长度无法解离"。这是对既有主张的修正,须在 ledger 中如作废条目一样明确记录。
- **PSI 相关低于 log2(L):** 更强的反向结果,处理同上,并需重新评估三分量框架中"负载门控"的表述。

**注意:仅 6 个数据点,任何相关系数都不稳健。** 报告时必须同时给出散点与样本量,不得仅以 r 值下结论。这一局限在 ledger §7 中已有记录(只有 3 个负载点),PSI 重做不改变该局限。

---

## 4. 执行计划

- **硬件:** RTX 4060 笔记本(本地),与批次 C/D 的 4090 无关。PSI 为纯前向推理,硬件差异不影响结果。
- **输入:** 主实验 full-branch `row` 的 checkpoint(5 seed × 2 block × 3 grid),以及批次 C channel-split `same_row_4` 的 checkpoint(d64 与 d256)。
- **不需要 GPU 训练,不与批次 D 冲突。**
- **顺序:** 先跑 `--at-init`(不需 checkpoint 权重,最快),再跑训练后模型。

---

## 5. 与批次 E 的关系

PSI 通过 P1(且 P2 结果已知并如实记录)是批次 E 开跑的**前置条件**。批次 E 的 E1 判据(负载匹配验证)将改写为基于 PSI 而非 `order_utilization`。

若 P1 不通过,批次 E 需重新设计负载匹配方案,不得在无可信负载轴的情况下开跑。
